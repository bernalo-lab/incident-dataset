ExplainError Calibration Project

Open Production Incident Dataset

A curated dataset of redacted real-world production failures used to study incident patterns, improve triage, and calibrate structured incident classification systems.

Why this exists

Modern systems generate large volumes of noisy and ambiguous errors. During incidents, engineers often have to decide quickly whether they are dealing with a dependency failure, timeout cascade, configuration issue, authentication problem, or something more subtle.

This dataset captures redacted real-world examples so engineers and tools can better recognise failure patterns and make faster, more structured triage decisions.

Core categories

API Failures

Upstream 503s, malformed responses, gateway errors.

Database Errors

Connection pool exhaustion, lock contention, query timeouts.

Payment Errors

Retry storms, duplicate authorisation attempts, gateway instability.

Timeout Patterns

Lambda timeouts, queue backlogs, slow downstream dependencies.

Authentication Errors

Expired tokens, failed refresh flows, 401 spikes.

Dependency Failures

DNS issues, unavailable upstream services, network resolution failures.

Configuration Errors

Bad endpoints, missing environment variables, secrets drift.

Cascading Failures

One slowdown triggers retries, queues, timeouts, and wider degradation.

Contribution principles

  • All examples must be fully redacted.
  • No company names, internal URLs, IDs, emails, or secrets.
  • Examples should be realistic, useful, and structured.
  • Misleading or ambiguous production errors are especially valuable.

Example record shape

{

  "category": "api-failures",
  "environment": "production",

  "error_message": "HTTP 503 Service Unavailable returned by upstream dependency",
  "log_excerpt": "POST /payments/authorize -> 503 returned from <payment-gateway>",
  "observed_symptoms": [
    "checkout requests failing",
    "retry attempts increasing rapidly"
  ],
  "expected_classification": "dependency/unavailable",
  "expected_severity": "high",
  "triage_notes": "Upstream payment gateway returned repeated 503 responses.",
  "confidence_hints": [
    "503 HTTP status code",
    "external service call"
  ],
  "action_signal": "retry_or_failover"
}
